# 风险管理

项目风险是指不确定性事件或条件，可能对项目目标（范围、进度、成本、质量）产生正面或负面影响。

## 风险管理过程

### 1. 规划风险管理
- 决定如何开展风险管理活动
- 制定风险管理计划

### 2. 识别风险
- 确定哪些风险可能影响项目
- 输出：风险登记册

### 3. 定性风险分析
- 评估风险发生概率和影响
- 确定风险优先级
- 输出：风险优先级矩阵

### 4. 定量风险分析
- 量化风险对项目整体目标的影响
- 常用技术：蒙特卡洛模拟、敏感性分析

### 5. 规划风险应对
- 制定应对策略
- 分配风险责任人

### 6. 实施风险应对
- 执行已批准的应对计划

### 7. 监督风险
- 跟踪已识别风险
- 监测残余风险
- 识别新风险

## 风险应对策略

### 负面风险（威胁）

| 策略 | 说明 | 适用场景 |
|------|------|---------|
| **规避** | 消除风险或保护目标 | 高概率、高影响风险 |
| **转移** | 将风险转移给第三方 | 财务风险、保险、外包 |
| **减轻** | 降低风险概率或影响 | 可控但无法消除的风险 |
| **接受** | 承认风险存在 | 低概率或低影响风险 |

### 正面风险（机会）

| 策略 | 说明 |
|------|------|
| **开拓** | 确保机会必然发生 |
| **分享** | 将机会转移给能更好利用的方 |
| **提高** | 增加机会概率或影响 |
| **接受** | 愿意利用但不强求 |

## 风险登记册内容

1. 已识别风险清单
2. 风险责任人
3. 风险应对计划
4. 风险触发因素
5. 残余风险和次生风险

## 风险概率与影响矩阵

|  | 低影响 | 中影响 | 高影响 |
|--|--------|--------|--------|
| **高概率** | 中 | 高 | 高 |
| **中概率** | 低 | 中 | 高 |
| **低概率** | 低 | 低 | 中 |

## 常用工具

- **SWOT 分析**：优势/劣势/机会/威胁
- **敏感性分析**：确定哪些风险影响最大
- **蒙特卡洛模拟**：定量分析项目工期/成本概率
