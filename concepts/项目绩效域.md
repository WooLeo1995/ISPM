# 项目绩效域

## 绩效域定义

绩效域是项目执行过程中需要关注的核心管理领域，通常分为两类：
- **项目管理绩效域**：范围、进度、成本、质量、资源、沟通、风险、采购、干系人
- **运营绩效域**：日常运营、持续运营

## 绩效域模型

```
        干系人
          ↑
      ┌───┴───┐
      ↓       ↓
    团队    交付←───范围
      ↓       ↑
      └───────┘
        绩效
```

## 关键绩效域

### 1. 项目干系人绩效域

| 要点 | 说明 |
|------|------|
| 识别干系人 | 全面识别 |
| 分析需求 | 理解期望 |
| 参与管理 | 引导支持 |

### 2. 团队绩效域

| 要点 | 说明 |
|------|------|
| 领导力 | 激励和引导 |
| 协作 | 团队合作 |
| 发展 | 能力提升 |

### 3. 交付绩效域

| 要点 | 说明 |
|------|------|
| 范围 | 完成全部工作 |
| 质量 | 满足需求 |
| 价值 | 实现预期效益 |

### 4. 规划绩效域

| 要点 | 说明 |
|------|------|
| 范围规划 | 确定边界 |
| 进度规划 | 排期 |
| 资源规划 | 配置 |
| 成本规划 | 预算 |

### 5. 绩效测量绩效域

| 要点 | 说明 |
|------|------|
| 偏差分析 | 实际 vs 计划 |
| 趋势分析 | 预测未来 |
| 挣值分析 | 综合绩效 |

## 绩效评估方法

| 方法 | 用途 |
|------|------|
| **KPI** | 关键指标评估 |
| **360度反馈** | 多维度评估 |
| **目标管理 (MBO)** | 目标对齐 |
| **OKR** | 目标与关键成果 |

## 绩效改进

```
计划 → 执行 → 检查 → 处理 (PDCA)
```

1. 收集数据
2. 分析差距
3. 制定改进计划
4. 实施改进
5. 验证效果
