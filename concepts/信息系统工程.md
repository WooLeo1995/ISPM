# 信息系统工程

## 信息系统的生命周期

```
规划 → 分析 → 设计 → 实现 → 运维 → 废弃
```

| 阶段 | 主要活动 |
|------|----------|
| **规划阶段** | 可行性研究、需求调研 |
| **分析阶段** | 业务分析、需求定义 |
| **设计阶段** | 系统设计、架构设计 |
| **实现阶段** | 编码、测试、部署 |
| **运维阶段** | 运营、维护 |
| **废弃阶段** | 数据归档、系统退役 |

## 软件工程

### 软件开发模型

| 模型 | 特点 | 适用场景 |
|------|------|----------|
| **瀑布模型** | 线性顺序 | 需求稳定 |
| **V模型** | 测试贯穿全程 | 硬件为主 |
| **增量模型** | 分阶段交付 | 需求逐步明确 |
| **迭代模型** | 反复完善 | 需求不确定 |
| **敏捷模型** | 快速迭代、拥抱变化 | 需求变化快 |

### 软件开发方法

| 方法 | 说明 |
|------|------|
| **结构化** | 自顶向下、模块化 |
| **面向对象** | 对象、类、继承 |
| **面向服务 (SOA)** | 服务重用 |
| **敏捷开发** | 迭代、协作 |

## 网络工程

### 网络系统建设

| 阶段 | 内容 |
|------|------|
| 网络规划 | 需求分析、拓扑设计 |
| 网络设计 | 设备选型、安全设计 |
| 网络实施 | 设备安装配置 |
| 网络测试 | 性能测试 |
| 网络运维 | 监控、维护 |

### 网络安全

| 层次 | 技术 |
|------|------|
| 物理层 | 隔离、冗余 |
| 网络层 | 防火墙、入侵检测 |
| 应用层 | 加密、身份认证 |

## 数据工程

### 数据生命周期

```
规划 → 采集 → 存储 → 处理 → 分析 → 展示 → 归档/销毁
```

### 数据治理

| 内容 | 说明 |
|------|------|
| 数据标准 | 格式、编码规则 |
| 数据质量 | 完整性、准确性 |
| 数据安全 | 访问控制、加密 |
| 元数据 | 描述数据的数据 |

## 面向对象设计

### 核心概念

| 概念 | 说明 |
|------|------|
| **类** | 抽象的数据类型 |
| **对象** | 类的实例 |
| **封装** | 隐藏内部实现 |
| **继承** | 子类复用父类 |
| **多态** | 同一接口不同实现 |
