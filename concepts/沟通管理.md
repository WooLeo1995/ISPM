# 沟通管理

## 沟通管理过程

### 1. 规划沟通管理
- 确定干系人信息需求
- 制定沟通计划

### 2. 管理沟通
- 创建和分发项目信息
- 确保信息有效传递

### 3. 监督沟通
- 监控沟通效果
- 调整沟通策略

## 沟通方式

| 方式 | 特点 | 适用场景 |
|------|------|----------|
| **推式沟通** | 主动推送 | 通知、报告 |
| **拉式沟通** | 按需获取 | 知识库、门户 |
| **交互式沟通** | 双向交流 | 会议、讨论 |

## 沟通渠道计算

**沟通渠道数 = n(n-1)/2**

其中 n = 干系人数量

| 干系人数 | 沟通渠道 |
|----------|----------|
| 3 | 3 |
| 5 | 10 |
| 10 | 45 |
| 20 | 190 |

> 干系人增加会导致沟通成本指数增长

## 会议分类

| 会议类型 | 目的 | 频率 |
|----------|------|------|
| **状态会议** | 同步进展 | 定期 |
| **站会** | 快速同步 | 每日 |
| **评审会议** | 评审成果 | 里程碑 |
| **回顾会议** | 总结改进 | 迭代末 |

## 沟通模型

```
发送者 → 编码 → 信息 → 解码 → 接收者
         ↑                    ↓
         ←── 反馈 ←───────────
```

### 沟通模型要素

| 要素 | 说明 |
|------|------|
| 编码 | 将想法转化为语言/文字 |
| 信息 | 编码后的内容 |
| 媒介 | 传递渠道（邮件、会议等） |
| 解码 | 接收者理解信息 |
| 反馈 | 确认理解是否正确 |

## 沟通障碍

| 障碍类型 | 示例 |
|----------|------|
| **发送障碍** | 表达不清、信息不完整 |
| **传递障碍** | 技术故障、信息丢失 |
| **接收障碍** | 选择性倾听、认知差异 |
| **环境障碍** | 噪音、干扰 |
| **文化障碍** | 语言差异、价值观不同 |

## 沟通技巧

1. 主动倾听
2. 确认理解
3. 注意非语言沟通
4. 适应受众
5. 选择合适媒介
