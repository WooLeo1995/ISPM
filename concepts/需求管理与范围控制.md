# 需求管理与范围控制

## 需求管理过程

### 1. 收集需求
- 访谈、问卷、头脑风暴
- 原型法、JAD研讨会
- 观察、业务文档分析

### 2. 定义需求
- 编写需求规格说明书
- 需求确认

### 3. 管理需求
- 变更控制
- 需求追踪

### 4. 验证需求
- 验收测试
- 确认满足业务目标

## 需求类型

| 类型 | 说明 |
|------|------|
| **业务需求** | 组织高层目标 |
| **用户需求** | 用户角度的期望 |
| **功能需求** | 系统应该做什么 |
| **非功能需求** | 性能、安全、可用性 |
| **接口需求** | 与其他系统的交互 |

## 需求追踪矩阵

| 属性 | 说明 |
|------|------|
| 需求ID | 唯一标识 |
| 需求描述 | 需求内容 |
| 来源 | 提出需求的干系人 |
| 优先级 | 高中低 |
| 状态 | 草稿/批准/实现/验证/关闭 |
| 版本 | 当前版本 |

## 需求追踪关系

```
业务目标 ← 用户需求 ← 功能需求 ← 设计 ← 编码 ← 测试
```

### 正向追踪

业务需求 → 产品功能 → 详细设计 → 实现 → 测试

### 反向追踪

测试 → 实现 → 设计 → 功能 → 业务需求

## 范围定义

### 项目范围 vs 产品范围

| 类型 | 说明 |
|------|------|
| **项目范围** | 完成项目所需的全部工作 |
| **产品范围** | 产品/服务具备的特性和功能 |

### 范围说明书

| 内容 | 说明 |
|------|------|
| 产品描述 | 最终成果 |
| 验收标准 | 交付物验收条件 |
| 可交付成果 | 明确列出 |
| 排除事项 | 明确不包括 |
| 约束条件 | 时间、资源限制 |
| 假设条件 | 假设为真的条件 |

## 范围控制

### 范围蔓延 (Scope Creep)

| 问题 | 后果 |
|------|------|
| 未经批准增加范围 | 进度延迟、成本超支 |
| 缺乏变更控制 | 项目失控 |
| 客户持续加需求 | 范围不断膨胀 |

### 控制措施

1. 严格执行变更流程
2. 定期审查范围
3. 明确验收标准
4. 区分镀金和必要功能

## WBS 创建

### 分解方法

| 方法 | 说明 |
|------|------|
| 按阶段 | 规划→执行→收尾 |
| 按产品 | 硬件+软件+文档 |
| 按子项目 | 分系统分解 |

### 分解步骤

1. 识别主要可交付成果
2. 确定分解方式
3. 分解到工作包
4. 分配编码
5. 验证分解
