# 制定项目章程

> 项目整合管理的第一个过程，也是项目的起点。
> 编写一份正式批准项目==并授权项目经理在项目活动中使用组织资源==的文件的过程。

## 📋 概述

| 属性         | 说明                                                     |
| ---------- | ------------------------------------------------------ |
| **所属过程组**  | 启动过程组                                                  |
| **所属知识领域** | 项目整合管理                                                 |
| **执行频次**   | ==仅开展一次或仅在项目的预定义时开展==                                  |
| **作用**     | 1. 明确项目与组织战略目标之间的直接联系<br>2. 确立项目的正式地位<br>3. 展示组织对项目的承诺 |

## 🔄 输入-工具与技术-输出 (ITTO)

```
┌─────────────────────────────────────────────────────────────────┐
│                        制定项目章程                              │
├─────────────────────────────────────────────────────────────────┤
│  输入                                                            │
│  ├── 立项管理文件（商业论证、成本效益分析）                       │
│  ├── 协议                                                        │
│  ├── 事业环境因素                                                │
│  └── 组织过程资产                                                │
├─────────────────────────────────────────────────────────────────┤
│  工具与技术                                                      │
│  ├── 专家判断                                                    │
│  ├── 数据收集（头脑风暴、焦点小组、访谈）                        │
│  ├── 人际关系与团队技能（冲突管理、引导）                       │
│  └── 会议                                                        │
├─────────────────────────────────────────────────────────────────┤
│  输出                                                            │
│  ├── 项目章程                                                    │
│  └── 假设日志                                                    │
└─────────────────────────────────────────────────────────────────┘
```

## 📥 输入详解

### 1. 立项管理文件

| 文件 | 说明 |
|------|------|
| **商业论证** | 文档化项目将为干系人带来的业务价值，包括成本效益分析 |
| **成本效益分析** | 评估项目的经济可行性，计算投资回报率（ROI）等指标 |

> 💡 商业论证是项目的"为什么"，项目章程是项目的"是什么"

### 2. 协议

- **定义**：为满足需求而与外部实体建立的书面协议
- **类型**：
  - 合同
  - 服务水平协议（SLA）
  - 谅解备忘录
  - 意向书

### 3. 事业环境因素

项目团队必须考虑的因素：
- 组织文化、行业标准或法规
- 市场条件、竞争态势
- 组织基础设施
- 政府或行业标准

### 4. 组织过程资产

可以帮助项目成功的资产：
- 组织的标准流程、政策
- 项目生命周期指南
- 模板（项目章程模板）
- 历史信息和经验教训

## 🔧 工具与技术详解

### 1. 专家判断

> 基于专业经验提供的指导

**应用场景：**
- 商业论证分析
- 项目目标定义
- 风险评估
- 资源估算

### 2. 数据收集技术

| 技术 | 说明 | 适用场景 |
|------|------|----------|
| **头脑风暴** | 快速生成大量想法 | 初期需求收集 |
| **焦点小组** | 召集干系人讨论特定主题 | 获取高质量反馈 |
| **访谈** | 一对一深入交流 | 关键干系人信息获取 |

### 3. 人际关系与团队技能

| 技能 | 说明 |
|------|------|
| **冲突管理** | 帮助干系人就目标、成功标准、高层级需求等达成一致 |
| **引导** | 有效引导团队活动成功达成决定、解决方案或结论 |
| **会议管理** | 准备议程、确保关键干系人代表参与、准备和发送会议纪要 |

### 4. 会议

> 与关键干系人开会，定义项目目的、目标和高层级需求

**关键产出：**
- 项目章程草案
- 干系人初步名单
- 高层级里程碑

## 📤 输出详解

### 1. 项目章程 ⭐⭐⭐

> **核心输出**：正式批准项目并授权项目经理使用组织资源的文件

#### 项目章程内容（11项）

| 序号 | 内容 | 说明 |
|------|------|------|
| 1 | **项目目的** | 项目存在的原因和价值 |
| 2 | **可测量的项目目标和成功标准** | 明确、可量化的目标 |
| 3 | **高层级需求** | 概要性的业务需求 |
| 4 | **高层级项目描述** | 项目的基本描述 |
| 5 | **边界定义** | 项目的范围边界 |
| 6 | **主要可交付成果** | 关键产出物清单 |
| 7 | **整体项目风险** | 重大风险概述 |
| 8 | **总体里程碑进度计划** | 关键时间节点 |
| 9 | **预先批准的财务资源** | 预算上限 |
| 10 | **关键干系人名单** | 主要决策者 |
| 11 | **项目审批要求** | 谁批准、评价标准 |
| 12 | **项目退出标准** | 什么条件下关闭项目 |
| 13 | **委派的项目经理及其职责和职权** | 授权范围 |
| 14 | **发起人或其他批准人员的姓名和职权** | 审批权限 |

**记忆口诀：** 目的目标需求要描述，风险进度财务干系人要审批，退出两个职权

#### 项目章程的关键作用

```
┌────────────────────────────────────────────────────────────┐
│                    项目章程的三大作用                       │
├────────────────────────────────────────────────────────────┤
│  1. 明确项目与组织战略目标之间的直接联系                    │
│     └── 确保项目对业务有实际贡献                            │
│                                                            │
│  2. 确立项目的正式地位                                     │
│     └── 项目正式成立，可调用组织资源                       │
│                                                            │
│  3. 展示组织对项目的承诺                                   │
│     └── 高层承诺，为项目提供支持                            │
└────────────────────────────────────────────────────────────┘
```

#### 重要知识点

> ⚠️ **项目经理的任命时机**
> 
> 应在规划开始之前任命项目经理，项目经理越早确认并任命越好，**最好在制定项目章程时就任命**
> 
> 项目章程授权项目经理进行项目管理过程中的规划、执行和控制

> ⚠️ **项目章程编制者**
> 
> 项目章程可由发起人编制，也可由项目经理与发起机构合作编制

> ⚠️ **项目正式启动标志**
> 
> 项目章程一旦被批准，就标志着项目的正式启动

### 2. 假设日志

| 内容 | 说明 |
|------|------|
| **定义** | 记录在项目启动期间编制的假设条件和制约因素 |
| **用途** | 整个项目生命周期中跟踪假设条件和制约因素 |
| **更新** | 随着项目进展发生变化时更新 |

## 🎯 核心考点

### 必背知识点

1. **项目章程的批准人**：发起人/高层
2. **项目章程授权给**：项目经理
3. **项目章程标志**：项目正式启动
4. **项目经理任命时机**：越早越好，最好在制定项目章程时

### 常见混淆点

| 区分 | 说明 |
|------|------|
| 项目章程 vs 项目管理计划 | 章程是"发起"文件，计划是"执行"文件 |
| 立项管理文件 vs 项目章程 | 立项文件是输入，项目章程是输出 |
| 项目章程 vs 合同 | 章程是内部文件，合同是外部协议 |

### 历年真题要点

- **23下3批**：过程1-制定项目章程的ITTO
- **23下2批**：项目章程内容（11项）
- **23上24**：人际关系与团队技能（冲突管理）
- **22上24**：项目管理计划和项目文件的区别
- **17上59**：项目经理应在规划开始之前任命
