# 整合管理

## 整合管理过程

### 1. 制定项目章程
- 正式批准项目
- 明确项目目标、范围、负责人
- 输出：项目章程

### 2. 制定项目管理计划
- 整合所有子计划
- 建立基准
- 输出：项目管理计划

### 3. 指导与管理项目工作
- 执行计划
- 创建可交付成果
- 输出：可交付成果、变更请求

### 4. 管理项目知识
- 整合知识资产
- 显性/隐性知识
- 输出：经验教训登记册

### 5. 监控项目工作
- 跟踪、审查、报告进展
- 偏差分析
- 输出：变更请求、工作绩效报告

### 6. 实施整体变更控制
- 审查变更请求
- 批准/拒绝变更
- 输出：批准的变更请求

### 7. 结束项目或阶段
- 验收可交付成果
- 总结经验教训
- 输出：最终产品/成果

## 关键输出

| 过程 | 主要输出 |
|------|----------|
| 制定项目章程 | 项目章程 |
| 制定管理计划 | 项目管理计划 |
| 指导执行 | 可交付成果、变更请求 |
| 管理知识 | 经验教训登记册 |
| 监控工作 | 变更请求、工作绩效报告 |
| 变更控制 | 批准的变更请求 |
| 结束项目 | 最终报告、经验教训 |

## 变更控制流程

```
变更请求提出
    ↓
变更影响分析
    ↓
CCB 审批
    ↓
    ├─ 批准 → 实施变更 → 验证
    └─ 拒绝 → 通知相关方
```

## 项目收尾

### 行政收尾

1. 确认所有交付成果完成
2. 验收可交付成果
3. 转移交付成果给客户
4. 完成财务结算
5. 记录经验教训
6. 释放资源

### 合同收尾

1. 确认合同条款完成
2. 验收最终交付
3. 关闭合同
4. 记录最终绩效

## 经验教训

- 贯穿项目全程
- 项目结束时归档
- 组织级过程资产
