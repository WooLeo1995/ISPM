# 高项备考重点

> 综合整理项目管理师考试核心知识点，涵盖项目管理、信息技术、治理法规等关键领域。

## 一、项目管理核心知识

### 1. 整合管理（必考）

**七大过程：**
1. 制定项目章程 - 输出项目章程
2. 制定管理计划 - 输出项目管理计划
3. 指导与管理项目工作 - 输出可交付成果
4. 管理项目知识 - 输出经验教训登记册
5. 监控项目工作 - 输出变更请求、工作绩效报告
6. 实施整体变更控制 - 输出批准的变更请求
7. 结束项目或阶段 - 输出最终产品/成果

**变更控制流程：**
```
变更请求 → 变更影响分析 → CCB审批
                         ├─ 批准 → 实施变更 → 验证
                         └─ 拒绝 → 通知相关方
```

### 2. 范围管理

| 过程 | 主要输出 |
|------|----------|
| 规划范围管理 | 范围管理计划、需求管理计划 |
| 收集需求 | 需求文件、需求跟踪矩阵 |
| 定义范围 | 项目范围说明书 |
| 创建WBS | 范围基准 |
| 确认范围 | 验收的可交付成果 |
| 控制范围 | 工作绩效信息 |

**WBS分解原则：**
- 100%原则：包含所有工作
- 80小时原则：工作包不超过80小时
- 滚动规划：近期详细，远期粗略

### 3. 进度管理

**关键路径法（CPM）：**

| 参数 | 说明 |
|------|------|
| ES | 最早开始时间 |
| EF | 最早完成时间 = ES + 工期 |
| LS | 最迟开始时间 = LF - 工期 |
| LF | 最迟完成时间 |
| 总浮动时间 | TF = LS - ES |
| 自由浮动时间 | 可延迟且不影响后置活动 |

**缩短工期方法：**
- 赶工（Crashing）：增加资源，成本增加
- 快速跟进（Fast Tracking）：并行执行，风险增加

**资源优化：**
- 资源平衡：解决冲突，可能延长工期
- 资源平滑：在浮动时间内调整

### 4. 成本管理

**挣值管理（EVM）：**

| 指标 | 英文 | 说明 |
|------|------|------|
| 计划价值 | PV | 应该完成工作的预算 |
| 挣值 | EV | 实际完成工作的预算 |
| 实际成本 | AC | 实际花费 |

**偏差分析：**
| 指标 | 公式 | 判断 |
|------|------|------|
| SV | EV - PV | >0超前，<0落后 |
| CV | EV - AC | >0节约，<0超支 |
| SPI | EV/PV | >1超前，<1落后 |
| CPI | EV/AC | >1节约，<1超支 |

**完工预测：**
| 公式 | 适用场景 |
|------|----------|
| EAC = BAC / CPI | 基于当前效率 |
| EAC = AC + (BAC - EV) | 基于原始计划 |
| EAC = AC + ETC | 重新估算 |

### 5. 质量管理

**三个过程：**
- 规划质量管理
- 实施质量保证（QA）
- 质量控制（QC）

**七种基本质量工具：**
| 工具 | 用途 |
|------|------|
| 因果图（鱼骨图） | 分析根本原因 |
| 流程图 | 展示过程步骤 |
| 核查表 | 收集数据 |
| 帕累托图 | 80/20法则，识别主要问题 |
| 直方图 | 显示数据分布 |
| 控制图 | 监控过程稳定性 |
| 散点图 | 展示两变量关系 |

**质量成本：**
| 类型 | 说明 |
|------|------|
| 预防成本 | 培训、流程改进 |
| 评估成本 | 测试、检查 |
| 内部失败成本 | 内部发现缺陷 |
| 外部失败成本 | 客户发现缺陷 |

### 6. 风险管理

**风险管理过程：**
1. 规划风险管理 - 输出风险管理计划
2. 识别风险 - 输出风险登记册
3. 定性风险分析 - 更新风险登记册
4. 定量风险分析 - 更新风险登记册
5. 规划风险应对 - 输出变更请求
6. 实施风险应对
7. 监督风险

**风险应对策略：**
| 类别 | 威胁策略 | 机会策略 |
|------|----------|----------|
| 上报 | 上报 | 上报 |
| 规避 | 消除威胁 | 消除机会 |
| 转移 | 保险、外包 | 伙伴关系 |
| 减轻 | 减少概率/影响 | 增加概率/影响 |
| 接受 | 应急储备 | 应急储备 |

## 三、IT治理与审计

### 1. IT治理框架

**管理层次：**
| 层次 | 职责 |
|------|------|
| 最高管理层 | IT战略与业务一致、价值交付、指导投资 |
| 执行管理层 | 目标制定、风险管理、核心竞争力 |
| 业务执行层 | 服务提供、基础设施维护、需求响应 |

### 2. IT审计风险

| 类型 | 说明 |
|------|------|
| 固有风险 | IT活动本身具有，无法控制 |
| 控制风险 | 与内部控制执行有效性有关 |
| 检查风险 | 审计程序未能发现重大问题 |
| 总体审计风险 | 各类风险的总和 |

### 3. 审计方法

- 访谈法、调查法、检查法、观察法、测试法
- 风险评估技术、审计抽样技术
- 计算机辅助审计（CAAT）、大数据审计

### 4. 审计证据特性

**充分性** - 数量要求
**客观性** - 事实材料
**相关性** - 与审计事项实质性联系
**可靠性** - 能反映经济活动特征
**合法性** - 法定种类、法定程序

## 五、法律法规与标准规范

### 1. 合同类型

| 类型 | 付款方式 | 风险 | 适用场景 |
|------|----------|------|----------|
| 总价合同 | 固定总价 | 低 | 范围明确 |
| 成本补偿合同 | 实报实销+利润 | 高 | 范围不确定 |
| 工料合同 | 单价×数量 | 中 | 范围模糊 |

### 2. 招标流程

1. 招标人编制招标文件
2. 发布招标公告或投标邀请书
3. 投标人编制并递交投标文件
4. 评标委员会评标
5. 招标人定标并公示
6. 发出中标通知书
7. 签订合同

### 3. 知识产权保护

- 著作权：作品完成即享有
- 专利权：发明、实用新型、外观设计
- 商标权：注册生效
- 软件著作权：登记保护

## 六、计算题专题

### 1. 挣值管理计算

```python
# 基本指标
PV = 计划价值
EV = 挣值
AC = 实际成本

# 偏差
SV = EV - PV
CV = EV - AC

# 绩效
SPI = EV / PV
CPI = EV / AC

# 预测
BAC = 完工预算
EAC = BAC / CPI  # 基于当前效率
VAC = BAC - EAC
```

### 2. 关键路径计算

```python
# 六个时间参数
ES = max(前导EF)
EF = ES + 工期
LF = min(后继LS)
LS = LF - 工期
TF = LS - ES
FF = 后继ES - 本活动EF
```

### 3. 决策树分析

```
期望货币价值(EMV) = Σ(结果值 × 概率)
选择EMV最高的方案
```

### 4. 现值计算

```
PV = FV / (1 + r)^n
NPV = Σ(PV) - 初始投资
```

### 5. 投资回收期

```
静态回收期 = 初始投资 / 年净现金流
```

### 项目管理49个过程（必背）

| 知识领域 | 过程数量 | 核心过程 |
|----------|----------|----------|
| 整合 | 7 | 制定章程、制定计划、监控、变更控制 |
| 范围 | 6 | 创建WBS、确认范围 |
| 进度 | 7 | 关键路径、进度压缩 |

| 质量 | 3 | 质量保证、质量控制 |
| 资源 | 6 | 团队建设、资源管理 |
| 沟通 | 4 | 管理沟通、监视沟通 |
| 风险 | 7 | 识别风险、定量分析、应对 |
| 采购 | 4 | 规划采购、实施采购 |
| 干系人 | 4 | 管理参与、监视参与 |

### 黄金知识点

1. **项目章程**：由发起人/高层批准，是项目与组织连结纽带
2. **范围基准**：包含范围说明书、WBS、WBS词典
3. **关键路径**：项目最短工期，浮动时间为0
4. **挣值管理**：SPI<0.9进度落后，CPI<0.9成本超支
5. **质量成本**：预防>失败，COPQ越低越好
6. **风险应对**：上报、规避、转移、减轻、接受
7. **CCB**：变更控制委员会，负责审批变更请求
8. **配置管理**：标识、控制、状态记录、审计

### 常见混淆点

| 区分 | 说明 |
|------|------|
| 质量保证QA vs 质量控制QC | QA过程审计，QC结果检查 |
| 关键路径 vs 关键链 | CPM确定型，CCM概率型+缓冲 |
| 风险 vs 问题 | 风险未发生，问题已发生 |
| 范围 vs 进度 vs 成本 | 黄金三角关系 |
| 确认范围 vs 核实可交付成果 | 客户验收 vs 内部检查 |
| 合同收尾 vs 行政收尾 | 合同完成 vs 项目完成 |
