## 第一部分：挣值管理（EVM）- 必考

### 核心概念

挣值管理是一种综合考虑**范围、进度、成本**的项目绩效测量方法。

### 三个基本指标

| 指标 | 英文 | 说明 | 计算 |
|------|------|------|------|
| **计划价值** | PV | 截至某时刻应该完成工作的预算价值 | 计划工作量 × 单价 |
| **挣值** | EV | 截至某时刻实际完成工作的预算价值 | 实际完成工作量 × 单价 |
| **实际成本** | AC | 截至某时刻实际花费的成本 | 实际支出 |

### 四个偏差指标

| 指标 | 公式 | 判断标准 | 记忆 |
|------|------|----------|------|
| **进度偏差** | SV = EV - PV | >0 超前，<0 落后 | EV 减 PV |
| **成本偏差** | CV = EV - AC | >0 节约，<0 超支 | EV 减 AC |
| **进度绩效指数** | SPI = EV / PV | >1 超前，<1 落后 | EV 除 PV |
| **成本绩效指数** | CPI = EV / AC | >1 节约，<1 超支 | EV 除 AC |

### 完工预测指标

| 指标 | 公式 | 说明 | 适用场景 |
|------|------|------|----------|
| **完工预算** | BAC | 项目总预算 | 已知 |
| **完工估算** | EAC | 预测项目完工成本 | 需要计算 |
| **完工偏差** | VAC = BAC - EAC | 预测成本与预算的差异 | 判断超支 |
| **完工尚需估算** | ETC = EAC - AC | 预计还需多少成本 | 剩余成本 |

### EAC 三种计算方法

#### 方法一：基于当前效率（最常用）

```
EAC = BAC / CPI

说明：假设未来的成本绩效与当前相同
适用：项目执行过程中，效率相对稳定
```

#### 方法二：基于原始计划

```
EAC = AC + (BAC - EV)

说明：假设未来按原始计划进行，不再有偏差
适用：当前偏差是一次性的，未来会改正
```

#### 方法三：重新估算

```
EAC = AC + ETC

说明：基于新的估算，完全重新评估
适用：项目情况发生重大变化
```

### EVM 计算题示例

#### 例题 1：基础计算

**题目**：
某项目总预算 BAC = 100 万元，计划工期 10 个月。
现在是第 5 个月，已完成工作的预算价值 EV = 40 万元，
计划完成工作的预算价值 PV = 50 万元，
实际成本 AC = 45 万元。

求：SV、CV、SPI、CPI、EAC（基于当前效率）

**解答**：

```
SV = EV - PV = 40 - 50 = -10 万元（进度落后 10 万元）
CV = EV - AC = 40 - 45 = -5 万元（成本超支 5 万元）
SPI = EV / PV = 40 / 50 = 0.8（进度落后 20%）
CPI = EV / AC = 40 / 45 = 0.889（成本超支 11.1%）
EAC = BAC / CPI = 100 / 0.889 = 112.5 万元
```

**判断**：
- 进度落后，SPI = 0.8 < 0.9（严重落后）
- 成本超支，CPI = 0.889 < 0.9（严重超支）
- 预计完工成本 112.5 万元，超支 12.5 万元

#### 例题 2：完工预测

**题目**：
某项目 BAC = 200 万元，当前 EV = 80 万元，AC = 100 万元，
PV = 100 万元。项目经理认为未来不会再有偏差。

求：EAC、VAC、ETC

**解答**：

```
CPI = EV / AC = 80 / 100 = 0.8
EAC = AC + (BAC - EV) = 100 + (200 - 80) = 220 万元
VAC = BAC - EAC = 200 - 220 = -20 万元（超支 20 万元）
ETC = EAC - AC = 220 - 100 = 120 万元（还需 120 万元）
```

### EVM 快速判断

| 状态 | SPI | CPI | 说明 |
|------|-----|-----|------|
| 优秀 | >1.0 | >1.0 | 进度超前，成本节约 |
| 良好 | 0.9~1.0 | 0.9~1.0 | 基本按计划进行 |
| 关注 | 0.8~0.9 | 0.8~0.9 | 需要采取措施 |
| 警告 | <0.8 | <0.8 | 严重偏差，需要纠正 |

## 第二部分：关键路径法（CPM）- 必考

### 核心概念

关键路径法用于确定项目**最长工期路径**，是项目进度管理的核心工具。

### 六个时间参数

| 参数 | 英文 | 说明 | 计算 |
|------|------|------|------|
| **最早开始** | ES | 活动最早能开始 | max(前导 EF) |
| **最早完成** | EF | 活动最早能完成 | ES + 工期 |
| **最迟开始** | LS | 活动最迟必须开始 | LF - 工期 |
| **最迟完成** | LF | 活动最迟必须完成 | min(后继 LS) |
| **总浮动时间** | TF | 可延迟时间 | LS - ES |
| **自由浮动时间** | FF | 不影响后续活动 | 后继 ES - 本活动 EF |

### 关键路径特征

- **关键路径**：TF = 0 的活动序列
- **关键活动**：关键路径上的活动
- **项目工期**：关键路径的总长度
- **浮动时间为 0**：任何延迟都影响项目

### CPM 计算题示例

#### 例题 1：基础计算

**题目**：
某项目有以下活动：

| 活动 | 工期 | 前导活动 |
|------|------|----------|
| A | 3 | - |
| B | 4 | A |
| C | 2 | A |
| D | 5 | B, C |
| E | 3 | D |

求：各活动的 ES、EF、LS、LF、TF，并找出关键路径

**解答**：

**第一步：计算 ES 和 EF（从前往后）**

```
A: ES = 0, EF = 0 + 3 = 3
B: ES = 3, EF = 3 + 4 = 7
C: ES = 3, EF = 3 + 2 = 5
D: ES = max(7, 5) = 7, EF = 7 + 5 = 12
E: ES = 12, EF = 12 + 3 = 15
```

项目工期 = 15 天

**第二步：计算 LF 和 LS（从后往前）**

```
E: LF = 15, LS = 15 - 3 = 12
D: LF = 12, LS = 12 - 5 = 7
B: LF = 7, LS = 7 - 4 = 3
C: LF = 7, LS = 7 - 2 = 5
A: LF = min(3, 5) = 3, LS = 3 - 3 = 0
```

**第三步：计算 TF**

```
A: TF = 0 - 0 = 0 ✓
B: TF = 3 - 3 = 0 ✓
C: TF = 5 - 3 = 2
D: TF = 7 - 7 = 0 ✓
E: TF = 12 - 12 = 0 ✓
```

**关键路径**：A → B → D → E（工期 15 天）

#### 例题 2：进度压缩

**题目**：
基于上题，如果要将项目工期从 15 天压缩到 12 天，应该压缩哪个活动？

**解答**：

只能压缩关键路径上的活动（A、B、D、E）。
- 压缩 A：最多压缩 3 天
- 压缩 B：最多压缩 4 天
- 压缩 D：最多压缩 5 天
- 压缩 E：最多压缩 3 天

需要压缩 3 天，可以：
- 压缩 D 3 天（从 5 天压缩到 2 天）
- 或压缩 A 1 天 + B 1 天 + D 1 天
- 或压缩 E 3 天

### CPM 快速计算技巧

1. **从前往后计算 ES 和 EF**
   - ES = max(所有前导活动的 EF)
   - EF = ES + 工期

2. **从后往前计算 LF 和 LS**
   - LF = min(所有后继活动的 LS)
   - LS = LF - 工期

3. **计算 TF**
   - TF = LS - ES
   - TF = 0 → 关键活动

4. **找关键路径**
   - 所有 TF = 0 的活动连接起来

## 第三部分：投资决策分析

### 投资回收期

#### 静态回收期

```
静态回收期 = 初始投资 / 年净现金流

说明：不考虑时间价值
适用：快速估算
```

#### 动态回收期

```
需要找到使累计现值 = 0 的时间点

说明：考虑时间价值
适用：精确计算
```

### 净现值（NPV）

```
NPV = Σ(现金流 / (1 + r)^n) - 初始投资

说明：
- NPV > 0：项目可行
- NPV < 0：项目不可行
- NPV 越大越好
```

### 投资回报率（ROI）

```
ROI = (收益 - 成本) / 成本 × 100%

说明：衡量投资效率
```

### 内部收益率（IRR）

```
IRR 是使 NPV = 0 的折现率

说明：
- IRR > 要求收益率：项目可行
- IRR < 要求收益率：项目不可行
```

### 投资决策计算题示例

#### 例题 1：投资回收期

**题目**：
某项目初始投资 100 万元，预期年净现金流 20 万元。
求：静态回收期

**解答**：

```
静态回收期 = 100 / 20 = 5 年
```

#### 例题 2：净现值

**题目**：
某项目初始投资 100 万元，预期 3 年内现金流分别为：
第 1 年：30 万元
第 2 年：40 万元
第 3 年：50 万元
折现率 r = 10%

求：NPV

**解答**：

```
NPV = 30/(1+0.1)^1 + 40/(1+0.1)^2 + 50/(1+0.1)^3 - 100
    = 30/1.1 + 40/1.21 + 50/1.331 - 100
    = 27.27 + 33.06 + 37.56 - 100
    = 97.89 - 100
    = -2.11 万元

结论：NPV < 0，项目不可行
```

## 第四部分：其他计算题

### 三点估算

#### 公式

```
期望值 = (乐观值 + 4×最可能值 + 悲观值) / 6

说明：
- 乐观值：最好情况
- 最可能值：最有可能发生
- 悲观值：最坏情况
```

#### 例题

**题目**：
某活动的三点估算为：
- 乐观值：5 天
- 最可能值：8 天
- 悲观值：14 天

求：期望工期

**解答**：

```
期望工期 = (5 + 4×8 + 14) / 6
         = (5 + 32 + 14) / 6
         = 51 / 6
         = 8.5 天
```

### 决策树分析

#### 公式

```
期望货币价值(EMV) = Σ(结果值 × 概率)

选择 EMV 最高的方案
```

#### 例题

**题目**：
某项目有两个方案：

方案 A：
- 成功概率 80%，收益 100 万元
- 失败概率 20%，损失 20 万元

方案 B：
- 成功概率 60%，收益 150 万元
- 失败概率 40%，损失 30 万元

求：选择哪个方案

**解答**：

```
方案 A 的 EMV = 100 × 0.8 + (-20) × 0.2 = 80 - 4 = 76 万元
方案 B 的 EMV = 150 × 0.6 + (-30) × 0.4 = 90 - 12 = 78 万元

选择方案 B（EMV 更高）
```

### 成本估算

#### 类比估算

```
新项目成本 = 历史项目成本 × 调整系数

说明：基于历史数据
```

#### 参数估算

```
成本 = 参数 × 单价

说明：基于参数和单价
```

#### 自下而上估算

```
总成本 = Σ(各工作包成本)

说明：最精确，但耗时
```

## 🎯 计算题做题技巧

### 1. 仔细读题

- 确认已知条件
- 确认求解目标
- 注意单位（万元、千元、元）

### 2. 列出公式

- 先写出需要用到的公式
- 确认公式正确
- 标注各个参数

### 3. 逐步计算

- 一步一步计算
- 保留中间结果
- 检查计算过程

### 4. 验证答案

- 检查数值是否合理
- 检查单位是否正确
- 检查符号是否正确

### 5. 总结结论

- 用文字说明结果含义
- 给出管理建议
- 说明项目状态

## 💡 常见错误

| 错误 | 原因 | 改正 |
|------|------|------|
| EVM 公式混淆 | 记忆不清 | 多做题，形成肌肉记忆 |
| CPM 计算错误 | 前后向计算混淆 | 分清方向，逐步计算 |
| 单位不统一 | 粗心 | 仔细检查单位 |
| 忘记考虑时间价值 | 概念不清 | 理解 NPV 的含义 |
| 关键路径找错 | 理解不深 | 确保 TF = 0 |

## 🚀 考试中的计算题策略

1. **先做简单题**：EVM 基础计算、三点估算
2. **再做中等题**：CPM 计算、投资决策
3. **最后做难题**：综合题、多步骤题

4. **时间分配**：
   - EVM 题：5-8 分钟
   - CPM 题：8-10 分钟
   - 投资决策题：5-7 分钟

5. **检查时间**：
   - 留出 5-10 分钟检查答案
   - 重点检查单位和符号
