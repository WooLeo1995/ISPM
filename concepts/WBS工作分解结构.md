# WBS 工作分解结构

WBS (Work Breakdown Structure) 是将项目全部工作按层级分解为可管理工作包的系统方法。

## 分解原则

| 原则 | 说明 |
|------|------|
| **100% 规则** | 包含所有项目工作，子项之和 = 父项 100% |
| **层级清晰** | 每层比上一层更细粒度 |
| **责任唯一** | 每个工作包有且只有一人负责 |
| **可交付成果导向** | 按产出而非行动分解 |

## WBS vs WBS 词典

| 项目 | WBS | WBS 词典 |
|------|-----|----------|
| 内容 | 分解结构图 | 每个工作包的详细说明 |
| 作用 | 展示层次关系 | 定义范围、负责人、验收标准 |

## 分解步骤

1. 识别项目主要交付物
2. 确定分解方式（按阶段/按交付物/混合）
3. 逐层分解
4. 分配编码
5. 验证分解完整性

## 层次结构示例

```
项目
├── 阶段1
│   ├── 工作包1.1
│   │   ├── 活动1.1.1
│   │   └── 活动1.1.2
│   └── 工作包1.2
├── 阶段2
│   └── ...
└── 项目管理
```

## 常见问题

| 问题 | 错误做法 | 正确做法 |
|------|---------|---------|
| 分解过细 | 列出所有日常任务 | 分解到可控的管理单元 |
| 分解不一致 | 不同子项目分解粒度不同 | 统一分解标准 |
| 无验收标准 | 只分解不定义 | 每个工作包必须有验收标准 |

## 范围基准

```
范围基准 = WBS + WBS词典 + 项目范围说明书
```

## 与范围说明书关系

- **项目范围说明书**：定义"做什么"和"不做什么"
- **WBS**：定义"怎么做"
