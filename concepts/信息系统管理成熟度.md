# 信息系统管理成熟度

## 能力成熟度模型 (CMM)

### 五个级别

| 级别 | 名称 | 特征 |
|------|------|------|
| 1 | 初始级 | 过程随意，成功依赖个人 |
| 2 | 可重复级 | 积累经验，建立基本流程 |
| 3 | 已定义级 | 过程标准化和文档化 |
| 4 | 量化管理级 | 过程度量和控制 |
| 5 | 持续优化级 | 过程改进和创新 |

## CMMI (能力成熟度模型集成)

### 两种表示

#### 连续式

| 能力等级 | 关注点 |
|----------|--------|
| CL 0 | 不完整 |
| CL 1 | 已执行 |
| CL 2 | 已管理 |
| CL 3 | 已定义 |
| CL 4 | 量化管理 |
| CL 5 | 持续优化 |

#### 阶段式

| 成熟度级别 | 等同于 |
|------------|--------|
| ML 1 | 初始 |
| ML 2 | 已管理 |
| ML 3 | 已定义 |
| ML 4 | 量化管理 |
| ML 5 | 优化 |

### 过程域

| 类别 | 过程域示例 |
|------|-----------|
| 过程管理 | OPD, OPF, OMD |
| 项目管理 | PP, PMC, SAM, RSKM |
| 工程 | REQM, RD, TS, PI |
| 支持 | CAR, DAR, MA |

## IT服务管理 (ITSM)

### ITIL 框架

| 版本 | 特点 |
|------|------|
| ITIL v1 | 基础，26个流程 |
| ITIL v2 | 合并为9个流程 |
| ITIL v3 | 生命周期方法 |
| ITIL 4 | 指导价值螺旋 |

### ITIL 4 核心概念

| 概念 | 说明 |
|------|------|
| **服务价值系统** | 价值创造体系 |
| **指导原则** | 决策基础 |
| **治理** | 战略到运营 |
| **服务价值链** | 价值流动 |
| **持续改进** | 不断优化 |

### ITIL 服务价值链

```
计划 → 准备 → 设计 → 转换 → 运营 → 改进
```

## 项目管理成熟度 (OPM3)

| 级别 | 特征 |
|------|------|
| 标准化 | 过程标准化 |
| 度量化 | 绩效可测量 |
| 控制化 | 主动控制 |
| 持续改进 | 优化创新 |

## 对比总结

| 模型 | 领域 | 关注点 |
|------|------|--------|
| CMM | 软件开发 | 过程改进 |
| CMMI |系统工程+软件 | 综合能力 |
| ITIL | IT服务 | 服务质量 |
| OPM3 | 项目组合 | 组织级项目管理 |
