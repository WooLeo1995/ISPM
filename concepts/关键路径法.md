# 关键路径法 (Critical Path Method)

关键路径法用于确定项目**最长工期路径**，是项目进度管理的核心工具。

## 基本概念

- **关键路径**：项目中耗时最长的活动序列
- **关键路径时长**：项目最短完工时间
- **关键活动**：关键路径上的活动，无浮动时间

## 六个时间参数

| 参数 | 说明 | 计算 |
|------|------|------|
| **ES** (Early Start) | 最早开始时间 | 前置活动 EF 的最大值 |
| **EF** (Early Finish) | 最早完成时间 | ES + 工期 |
| **LS** (Late Start) | 最迟开始时间 | LF - 工期 |
| **LF** (Late Finish) | 最迟完成时间 | 后置活动 LS 的最小值 |
| **总浮动时间** | TF = LS - ES | 活动可延迟但不影响完工时间 |
| **自由浮动时间** | FF | 活动可延迟且不影响后置活动 |

## 关键路径特征

1. 关键路径可能有多条
2. 关键路径随项目进展可能变化
3. 压缩关键路径是缩短工期的唯一方法
4. 关键路径上的活动浮动时间通常为 0

## 缩短工期的手段

| 方法 | 说明 |
|------|------|
| **赶工 (Crashing)** | 增加资源换取时间，成本增加 |
| **快速跟进 (Fast Tracking)** | 并行执行顺序活动，风险增加 |
| **减少关键活动工期** | 投入更多资源/优化方法 |

## 资源优化

- **资源平衡**：解决资源冲突，可能延长工期
- **资源平滑**：在浮动时间内调整资源，不延长工期

## 与关键链区别

| 概念 | 关键路径法 (CPM) | 关键链法 (CCM) |
|------|-----------------|----------------|
| 基础 | 确定型估算 | 概率型估算 |
| 缓冲 | 无 | 项目缓冲、接驳缓冲 |
| 关注 | 活动逻辑 | 资源约束 |
