# IT治理与COBIT框架

## IT治理

IT治理关注：
- IT与业务一致性
- IT价值交付
- 风险管理
- 资源管理
- 绩效衡量

## COBIT框架

COBIT (Control Objectives for Information and Related Technologies) 是ISACA发布的IT治理与管理框架。

### 五个关键领域

| 领域 | 说明 |
|------|------|
| **EDM** (Evaluate, Direct, Monitor) | 治理：评估、指导、监控 |
| **APO** (Align, Plan, Organize) | 管理：规划与组织 |
| **BAI** (Build, Acquire, Implement) | 管理：构建与获取 |
| **DSS** (Deliver, Service, Support) | 管理：交付与服务 |
| **MEA** (Monitor, Evaluate, Assess) | 管理：监控与评估 |

### 治理 vs 管理

| 维度 | 治理 | 管理 |
|------|------|------|
| 关注点 | 长期目标、风险偏好 | 短期执行、效率 |
| 活动 | 评估、指导、监控 | 计划、构建、运营 |
| 层级 | 高层 | 中层/执行层 |
| 框架 | EDM | APO/BAI/DSS/MEA |

## Goals Cascade

```
企业目标
    ↓
IT目标
    ↓
治理目标 (EDM)
    ↓
管理目标 (APO/BAI/DSS/MEA)
```

## IT治理主要任务

1. **战略一致性** — IT战略与企业战略对齐
2. **价值交付** — 确保IT带来预期价值
3. **风险管理** — 保护IT资产
4. **资源管理** — 优化IT资源
5. **绩效评估** — 衡量IT绩效
